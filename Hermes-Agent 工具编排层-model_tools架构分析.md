# Hermes-Agent 工具编排层 (model_tools.py) 架构分析

## 1. 系统概述

`model_tools.py` 是 Hermes-Agent 的**工具编排层**，作为工具注册表 (`tools.registry`) 的薄封装层，负责：

1. **工具发现与注册**：导入所有工具模块，触发自注册机制
2. **Schema 过滤与组装**：根据启用的工具集，生成符合 OpenAI 格式的工具定义列表
3. **异步桥接**：提供同步到异步的桥接机制，支持异步工具处理器
4. **函数调用分发**：接收 LLM 的函数调用请求，路由到对应的工具处理器
5. **参数类型强制转换**：将 LLM 返回的字符串参数转换为正确的 JSON Schema 类型
6. **插件扩展**：支持 MCP 工具和插件工具的动态发现

### 1.1 核心功能特性

| 功能模块 | 描述 |
|---------|------|
| **工具发现** | 导入所有工具模块，触发 `registry.register()` 自注册 |
| **Schema 过滤** | 根据 enabled/disabled_toolsets 过滤工具定义 |
| **异步桥接** | 持久化事件循环，避免 "Event loop is closed" 错误 |
| **参数强制转换** | 将字符串 "42" → 42，"true" → true |
| **动态 Schema 重建** | 根据可用工具重建 execute_code 和 browser_navigate 的 Schema |
| **插件支持** | MCP 工具发现和插件钩子调用 |
| **遗留工具集兼容** | 支持旧版 `_tools` 后缀的工具集名称 |

### 1.2 架构设计原则

1. **薄封装层**：不直接实现工具逻辑，而是通过 registry 委托
2. **持久化事件循环**：避免频繁创建/销毁事件循环导致的客户端错误
3. **线程安全**：主线程和工作线程各自拥有独立的事件循环
4. **参数类型安全**：LLM 经常返回字符串类型的数字和布尔值，需要强制转换
5. **动态 Schema**：根据实际可用的工具动态调整 Schema 描述，避免幻觉调用

### 2.2 异步桥接架构图

```
┌──────────────────────────────────────────────────────────────────────────────┐
│                          异步桥接架构 (Async Bridging)                         │
│                                                                              │
│   问题：asyncio.run() 每次创建并关闭新的事件循环，导致缓存的 httpx/            │
│         AsyncOpenAI 客户端在垃圾回收时触发 "Event loop is closed" 错误        │
│                                                                              │
│   解决方案：使用持久化事件循环，保持循环生命周期与线程生命周期一致              │
│                                                                              │
│  ┌──────────────────────────────────────────────────────────────────────┐   │
│  │  主线程 (Main Thread)                                                │   │
│  │                                                                      │   │
│  │  _tool_loop = None                                                   │   │
│  │  _tool_loop_lock = threading.Lock()                                  │   │
│  │                                                                      │   │
│  │  _get_tool_loop():                                                   │   │
│  │    if _tool_loop is None or _tool_loop.is_closed():                  │   │
│  │        _tool_loop = asyncio.new_event_loop()                         │   │
│  │    return _tool_loop                                                 │   │
│  │                                                                      │   │
│  │  用途：CLI 模式、Gateway 同步调用                                    │   │
│  └──────────────────────────────────┬───────────────────────────────────┘   │
│                                     │                                        │
│                                     ▼                                        │
│  ┌──────────────────────────────────────────────────────────────────────┐   │
│  │  工作线程 (Worker Thread)                                            │   │
│  │                                                                      │   │
│  │  _worker_thread_local = threading.local()                            │   │
│  │                                                                      │   │
│  │  _get_worker_loop():                                                 │   │
│  │    loop = getattr(_worker_thread_local, 'loop', None)                │   │
│  │    if loop is None or loop.is_closed():                              │   │
│  │        loop = asyncio.new_event_loop()                               │   │
│  │        asyncio.set_event_loop(loop)                                  │   │
│  │        _worker_thread_local.loop = loop                              │   │
│  │    return loop                                                       │   │
│  │                                                                      │   │
│  │  用途：delegate_task 的 ThreadPoolExecutor 并发执行                  │   │
│  └──────────────────────────────────┬───────────────────────────────────┘   │
│                                     │                                        │
│                                     ▼                                        │
│  ┌──────────────────────────────────────────────────────────────────────┐   │
│  │  _run_async(coro) 分发逻辑                                           │   │
│  │                                                                      │   │
│  │  1. 检查当前是否有运行中的事件循环                                    │   │
│  │     • 有 (Gateway/RL 环境) → 在新线程中运行 asyncio.run()            │   │
│  │     • 无 → 继续下一步                                                │   │
│  │                                                                      │   │
│  │  2. 检查是否在工作线程                                              │   │
│  │     • 是 → 使用 _get_worker_loop()                                  │   │
│  │     • 否 (主线程) → 使用 _get_tool_loop()                           │   │
│  │                                                                      │   │
│  │  3. 在选定的持久化循环上运行协程                                    │   │
│  │     • loop.run_until_complete(coro)                                 │   │
│  └──────────────────────────────────────────────────────────────────────┘   │
│                                                                              │
└──────────────────────────────────────────────────────────────────────────────┘
```

### 4.1 异步桥接实现

```python
# 主线程持久化事件循环
_tool_loop = None
_tool_loop_lock = threading.Lock()

def _get_tool_loop():
    """返回主线程的长生命周期事件循环"""
    global _tool_loop
    with _tool_loop_lock:
        if _tool_loop is None or _tool_loop.is_closed():
            _tool_loop = asyncio.new_event_loop()
        return _tool_loop

# 工作线程持久化事件循环 (线程本地存储)
_worker_thread_local = threading.local()

def _get_worker_loop():
    """返回当前工作线程的长生命周期事件循环"""
    loop = getattr(_worker_thread_local, 'loop', None)
    if loop is None or loop.is_closed():
        loop = asyncio.new_event_loop()
        asyncio.set_event_loop(loop)
        _worker_thread_local.loop = loop
    return loop

# 统一的异步桥接入口
def _run_async(coro):
    """从同步上下文运行异步协程"""
    try:
        loop = asyncio.get_running_loop()
    except RuntimeError:
        loop = None

    if loop and loop.is_running():
        # 已在异步上下文 (Gateway/RL) → 在新线程运行
        with concurrent.futures.ThreadPoolExecutor(max_workers=1) as pool:
            future = pool.submit(asyncio.run, coro)
            return future.result(timeout=300)

    if threading.current_thread() is not threading.main_thread():
        # 工作线程 → 使用线程本地循环
        worker_loop = _get_worker_loop()
        return worker_loop.run_until_complete(coro)

    # 主线程 → 使用共享循环
    tool_loop = _get_tool_loop()
    return tool_loop.run_until_complete(coro)
```

**设计要点：**
- **持久化循环**：避免 `asyncio.run()` 的创建-销毁周期
- **线程隔离**：主线程和工作线程各自拥有独立循环，避免竞争
