## 🔍 日志分析步骤


## 🔍 常见问题排查

### 问题 1: 文件没有创建

```bash
# 检查日志中是否有错误
grep -E "ERROR|Failed|Error" ~/.hermes/logs/errors.log

# 检查文件工具是否执行
grep "write_file" ~/.hermes/logs/agent.log

# 检查文件路径
grep "Writing to:" ~/.hermes/logs/agent.log | tail -1
```

### 问题 2: 工具调用失败

```bash
# 查看工具执行错误
grep "Tool.*failed\|Tool.*error" ~/.hermes/logs/errors.log

# 查看工具执行详情
grep -A 5 "Executing tool: write_file" ~/.hermes/logs/agent.log
```

### 问题 3: API 调用失败

```bash
# 查看 API 错误
grep -E "API.*error|Rate limit|Timeout" ~/.hermes/logs/errors.log

# 查看 LLM 调用详情
grep "Calling LLM:" ~/.hermes/logs/agent.log | tail -5
```

### 问题 4: 找不到日志

```bash
# 检查日志文件是否存在
ls -lh ~/.hermes/logs/

# 检查日志级别
grep "level:" ~/.hermes/config.yaml

# 如果是 INFO，改为 DEBUG
hermes config set logging.level DEBUG
```
