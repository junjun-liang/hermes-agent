# model_tools.py 架构与业务逻辑分析

## 1. 模块概述

`model_tools.py` 是 Hermes-Agent 的工具编排核心模块，作为工具注册表（`tools.registry`）之上的精简编排层。该模块负责工具发现、Schema过滤、参数类型转换、工具调用分发等功能，是连接LLM工具调用请求与实际工具实现的桥梁。

### 1.1 核心职责

| 功能模块 | 描述 |
|---------|------|
| **工具发现** | 导入所有工具模块，触发 `registry.register()` 注册 |
| **Schema提供** | `get_tool_definitions()` 根据工具集过滤返回OpenAI格式的Schema |
| **调用分发** | `handle_function_call()` 路由LLM的工具调用请求 |
| **异步桥接** | 管理持久化事件循环，解决GC清理问题 |
| **参数转换** | 自动将字符串参数转换为Schema声明的类型 |
| **插件集成** | 支持MCP工具、插件工具的发现和钩子调用 |

### 1.2 公开API

```python
get_tool_definitions(enabled_toolsets, disabled_toolsets, quiet_mode) -> list
handle_function_call(function_name, function_args, task_id, user_task) -> str
TOOL_TO_TOOLSET_MAP: dict
TOOLSET_REQUIREMENTS: dict
get_all_tool_names() -> list
get_toolset_for_tool(name) -> str
get_available_toolsets() -> dict
check_toolset_requirements() -> dict
check_tool_availability(quiet) -> tuple
```

### 4.1 工具发现机制

_modules = [

**设计要点：**

- 异常捕获确保单个工具加载失败不影响其他工具

### 4.2 异步桥接机制

```python
# 主线程持久化循环
_tool_loop = None
_tool_loop_lock = threading.Lock()

def _get_tool_loop():
    global _tool_loop
    with _tool_loop_lock:
        if _tool_loop is None or _tool_loop.is_closed():
            _tool_loop = asyncio.new_event_loop()
        return _tool_loop

# 工作线程每线程循环
_worker_thread_local = threading.local()

def _get_worker_loop():
    loop = getattr(_worker_thread_local, 'loop', None)
    if loop is None or loop.is_closed():
        loop = asyncio.new_event_loop()
        asyncio.set_event_loop(loop)
        _worker_thread_local.loop = loop
    return loop
```

**设计要点：**
- 防止 "Event loop is closed" 错误
- 缓存的 httpx/AsyncOpenAI 客户端绑定到活跃循环
- 线程安全，支持并发工具调用

## 6. 设计模式

### 6.1 适配器模式

`model_tools.py` 作为工具注册表和调用方之间的适配层，将 registry 的底层API转换为上层需要的格式。

### 6.2 策略模式

参数类型转换使用策略模式，根据Schema声明的类型选择不同的转换策略（整数、浮点数、布尔值、联合类型）。

### 6.3 工厂模式

`_run_async()` 根据运行环境（主线程、工作线程、异步上下文）选择不同的事件循环创建策略。

### 6.4 责任链模式

工具调用通过插件钩子（pre_tool_call / post_tool_call）形成责任链，允许插件拦截或修改调用行为。

## 8. 总结

`model_tools.py` 是 Hermes-Agent 的工具编排核心，主要职责包括：

1. **工具发现**: 导入模块触发注册，支持MCP和插件工具
2. **Schema提供**: 根据工具集过滤返回OpenAI格式的工具定义
3. **参数转换**: 自动将字符串参数转换为Schema声明的类型
4. **调用分发**: 路由LLM的工具调用请求到实际实现
5. **异步桥接**: 管理持久化事件循环，解决GC清理问题

该模块设计精巧，通过持久化事件循环、动态Schema构建、插件钩子等机制，确保了工具调用的稳定性、灵活性和可扩展性。
