## 设计灵感

**不要创建无聊的幻灯片。** 白色背景上的普通项目符号不会给人留下深刻印象。考虑为每张幻灯片从这个列表中获取灵感。

### 开始之前

- **选择大胆的、基于内容的调色板**：调色板应该感觉是为这个主题设计的。如果将你的颜色换到完全不同的演示文稿中仍然"有效"，说明你的选择不够具体。
- **主导胜于平等**：一种颜色应占主导（60-70% 视觉权重），配 1-2 种支持色调和一种鲜明点缀。绝不要给所有颜色平等的权重。
- **深色/浅色对比**：标题和结论幻灯片使用深色背景，内容使用浅色（"三明治"结构）。或者全程使用深色以获得高级感。
- **坚持视觉主题**：选择一个独特的元素并重复使用——圆角图像框架、彩色圆形中的图标、粗单侧边框。在每张幻灯片上保持一致。

### 调色板

选择与主题匹配的颜色——不要默认使用通用蓝色。使用这些调色板作为灵感：

| 主题 | 主色 | 辅色 | 点缀色 |
|-------|---------|-----------|--------|
| **午夜高管** | `1E2761`（海军蓝） | `CADCFC`（冰蓝） | `FFFFFF`（白色） |
| **森林与苔藓** | `2C5F2D`（森林绿） | `97BC62`（苔藓绿） | `F5F5F5`（奶油） |
| **珊瑚能量** | `F96167`（珊瑚红） | `F9E795`（金色） | `2F3C7E`（海军蓝） |
| **温暖赤陶** | `B85042`（赤陶） | `E7E8D1`（沙色） | `A7BEAE`（鼠尾草） |
| **海洋渐变** | `065A82`（深蓝） | `1C7293`（青绿） | `21295C`（午夜） |
| **木炭极简** | `36454F`（木炭灰） | `F2F2F2`（灰白） | `212121`（黑色） |
| **青色信任** | `028090`（青色） | `00A896`（海沫） | `02C39A`（薄荷） |
| **浆果与奶油** | `6D2E46`（浆果） | `A26769`（灰玫瑰） | `ECE2D0`（奶油） |
| **鼠尾草宁静** | `84B59F`（鼠尾草） | `69A297`（桉树） | `50808E`（石板） |
| **樱桃大胆** | `990011`（樱桃红） | `FCF6F5`（灰白） | `2F3C7E`（海军蓝） |

### 每张幻灯片

**每张幻灯片都需要视觉元素**——图像、图表、图标或形状。纯文本幻灯片令人遗忘。

**布局选项：**
- 两列（左文本，右插图）
- 图标 + 文本行（彩色圆形中的图标，粗标题，下方描述）
- 2x2 或 2x3 网格（一侧图像，另一侧内容块网格）
- 半出血图像（全左侧或右侧）带内容叠加

**数据展示：**
- 大型数据标注（60-72pt 大数字，下方小标签）
- 比较列（之前/之后、优点/缺点、并排选项）
- 时间线或流程（编号步骤、箭头）

**视觉润色：**
- 章节标题旁小彩色圆形中的图标
- 关键数据或标语的斜体点缀文本

### 字体排版

**选择有趣的字体搭配**——不要默认使用 Arial。选择有个性的标题字体，搭配干净的正文字体。

| 标题字体 | 正文字体 |
|-------------|-----------|
| Georgia | Calibri |
| Arial Black | Arial |
| Calibri | Calibri Light |
| Cambria | Calibri |
| Trebuchet MS | Calibri |
| Impact | Arial |
| Palatino | Garamond |
| Consolas | Calibri |

| 元素 | 大小 |
|---------|------|
| 幻灯片标题 | 36-44pt 粗体 |
| 章节标题 | 20-24pt 粗体 |
| 正文 | 14-16pt |
| 图注 | 10-12pt 柔和 |

### 间距

- 最小边距 0.5"
- 内容块之间 0.3-0.5"
- 留出呼吸空间——不要填满每一寸

### 避免（常见错误）

- **不要重复相同布局**——在幻灯片之间变化列、卡片和标注
- **不要居中正文**——左对齐段落和列表；仅居中标题
- **不要吝啬大小对比**——标题需要 36pt+ 才能从 14-16pt 正文中脱颖而出
- **不要默认蓝色**——选择反映具体主题的颜色
- **不要随机混合间距**——选择 0.3" 或 0.5" 间距并一致使用
- **不要只修饰一张幻灯片而其余留白**——要么完全投入，要么全程保持简单
- **不要创建纯文本幻灯片**——添加图像、图标、图表或视觉元素；避免纯标题 + 项目符号
- **不要忘记文本框内边距**——当将线条或形状与文本边缘对齐时，在文本框上设置 `margin: 0` 或偏移形状以考虑内边距
- **不要使用低对比度元素**——图标和文本都需要与背景形成强对比；避免浅色背景上的浅色文本或深色背景上的深色文本
- **绝不要在标题下使用点缀线**——这是 AI 生成幻灯片的标志；改用空白或背景色

## 质量保证（必需）

**假设有问题。你的工作是找到它们。**

你的第一次渲染几乎从不正确。将质量保证视为猎杀 bug，而非确认步骤。如果第一次检查中你没有发现任何问题，说明你看得不够仔细。

### 内容质量保证

```bash
python -m markitdown output.pptx
```

检查缺失内容、拼写错误、顺序错误。

**使用模板时，检查是否有残留占位符文本：**

```bash
python -m markitdown output.pptx | grep -iE "xxxx|lorem|ipsum|this.*(page|slide).*layout"
```

如果 grep 返回结果，在宣布成功之前修复它们。

### 视觉质量保证

**⚠️ 使用子 Agent**——即使只有 2-3 张幻灯片。你一直盯着代码看，会看到你期望的而非实际存在的内容。子 Agent 有新鲜的眼睛。

将幻灯片转换为图像（参见[转换为图像](#converting-to-images)），然后使用此提示：

```
视觉检查这些幻灯片。假设有问题——找到它们。

查找：
- 重叠元素（文本穿过形状、线条穿过单词、堆叠元素）
- 文本溢出或在边缘/框边界处被截断
- 为单行文本定位的装饰线，但标题换行为两行
- 源引用或页脚与上方内容冲突
- 元素太近（< 0.3" 间距）或卡片/部分几乎接触
- 不均匀的间距（一处大面积空白，另一处拥挤）
- 距幻灯片边缘边距不足（< 0.5"）
- 列或相似元素未一致对齐
- 低对比度文本（如奶油色背景上的浅灰色文本）
- 低对比度图标（如深色背景上没有对比圆形的深色图标）
- 文本框太窄导致过度换行
- 残留占位内容

对于每张幻灯片，列出问题或关注区域，即使是次要的。

读取并分析这些图像：
1. /path/to/slide-01.jpg（预期：[简要描述]）
2. /path/to/slide-02.jpg（预期：[简要描述]）

报告发现的所有问题，包括次要问题。
```

### 验证循环

1. 生成幻灯片 → 转换为图像 → 检查
2. **列出发现的问题**（如果没有找到，更批判性地再看一次）
3. 修复问题
4. **重新验证受影响的幻灯片**——一个修复经常引发另一个问题
5. 重复直到完整一轮不发现新问题

**在完成至少一次修复-验证循环之前，不要宣布成功。**

## 转换为图像

将演示文稿转换为单独幻灯片图像以进行视觉检查：

```bash
python scripts/office/soffice.py --headless --convert-to pdf output.pptx
pdftoppm -jpeg -r 150 output.pdf slide
```

这将创建 `slide-01.jpg`、`slide-02.jpg` 等。

修复后重新渲染特定幻灯片：

```bash
pdftoppm -jpeg -r 150 -f N -l N output.pdf slide-fixed
```
