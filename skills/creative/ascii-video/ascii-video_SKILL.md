# ASCII 视频制作流水线

## 创意标准

这是视觉艺术。ASCII 字符是媒介；电影是标准。

**在编写任何代码之前**，阐明创意概念。情绪是什么？这讲述了什么视觉故事？这个作品与其他 ASCII 视频有何不同？用户的提示是起点——用创意野心解读它，而非字面转录。

**首次渲染的卓越表现是不可妥协的。** 输出必须在不需要修订轮次的情况下具有视觉冲击力。如果某些内容看起来通用、平淡或像"AI 生成的 ASCII 艺术"，那就是错误的——在交付之前重新思考创意概念。

**超越参考词汇。** 参考资料中的效果目录、着色器预设和调色板库是起始词汇。对于每个项目，结合、修改和发明新图案。目录是颜料调色板——你创作绘画。

**主动创意。** 当项目需要时扩展技能的词汇。如果参考资料没有愿景所需的内容，就构建它。包含至少一个用户没有要求但会欣赏的视觉时刻——一个过渡、一个效果、一个提升整体作品的颜色选择。

**美学统一性高于技术正确性。** 视频中的所有场景必须通过统一的视觉语言连接——共享色温、相关字符调色板、一致的运动词汇。每个场景使用随机不同效果的技术正确视频在美学上是失败的。

**密集、分层、经过深思熟虑。** 每一帧都应该值得观看。绝不要纯黑色背景。始终使用多网格构图。始终按场景变化。始终有意使用颜色。

## 技术栈

每个项目使用单个自包含的 Python 脚本。不需要 GPU。

| 层 | 工具 | 用途 |
|-------|------|---------|
| 核心 | Python 3.10+, NumPy | 数学、数组运算、向量化效果 |
| 信号 | SciPy | FFT、峰值检测（音频模式） |
| 图像 | Pillow (PIL) | 字体光栅化、帧解码、图像 I/O |
| 视频 I/O | ffmpeg (CLI) | 解码输入、编码输出、混合音频 |
| 并行 | concurrent.futures | N 个工作线程用于批量/片段渲染 |
| TTS | ElevenLabs API（可选） | 生成旁白片段 |
| 可选 | OpenCV | 视频帧采样、边缘检测 |

## 流水线架构

每种模式都遵循相同的 6 阶段流水线：

```
INPUT → ANALYZE → SCENE_FN → TONEMAP → SHADE → ENCODE
```

1. **INPUT**——加载/解码源素材（视频帧、音频样本、图像或无）
2. **ANALYZE**——提取每帧特征（音频频段、视频亮度/边缘、运动向量）
3. **SCENE_FN**——场景函数渲染到像素画布（`uint8 H,W,3`）。通过 `_render_vf()` + 像素混合模式合成多个字符网格。参见 `references/composition.md`
4. **TONEMAP**——基于百分位数的自适应亮度归一化。参见 `references/composition.md` § 自适应色调映射
5. **SHADE**——通过 `ShaderChain` + `FeedbackBuffer` 进行后处理。参见 `references/shaders.md`
6. **ENCODE**——将原始 RGB 帧通过管道传输到 ffmpeg 进行 H.264/GIF 编码

### 每节变化

绝不要对整个视频使用相同的配置。对于每个部分/场景：
- **不同的背景效果**（或组合 2-3 个）
- **不同的字符调色板**（匹配情绪）
- **不同的颜色策略**（或至少不同的色调）
- **变化着色器强度**（峰值期间更多泛光，安静期间更多颗粒）
- **如果粒子激活，使用不同的粒子类型**

### 项目特定创新

对于每个项目，至少发明一个：
- 匹配主题的自定义字符调色板
- 自定义背景效果（组合/修改现有构建块）
- 自定义颜色调色板（匹配品牌/情绪的离散 RGB 集）
- 自定义粒子字符集
- 新颖的场景过渡或视觉时刻

不要只是从目录中选择。目录是词汇——你写诗。

## 工作流

### 步骤 1：创意愿景

在任何代码之前，阐明创意概念：

- **情绪/氛围**：观众应该感受到什么？充满活力、冥想、混乱、优雅、不祥？
- **视觉故事**：在持续时间内发生了什么？建立紧张感？转变？溶解？
- **颜色世界**：暖/冷？单色？霓虹？大地色调？主色调是什么？
- **字符纹理**：密集数据？稀疏星星？有机点？几何块？
- **这个有何不同**：让这个作品独特的一件事是什么？
- **情感弧线**：场景如何进展？以能量开场，建立到高潮，解决？

将用户的提示映射到美学选择。"chill lo-fi visualizer"与"glitch cyberpunk data stream"的要求完全不同。

### 步骤 2：技术设计

- **模式**——上述 6 种模式中的哪一种
- **分辨率**——横向 1920x1080（默认），纵向 1080x1920，方形 1080x1080 @ 24fps
- **硬件检测**——自动检测核心/内存，设置质量配置。参见 `references/optimization.md`
- **部分**——将时间戳映射到场景函数，每个都有自己的效果/调色板/颜色/着色器配置
- **输出格式**——MP4（默认）、GIF（640x360 @ 15fps）、PNG 序列

### 步骤 3：构建脚本

单个 Python 文件。组件（带参考）：

1. **硬件检测 + 质量配置**——`references/optimization.md`
2. **输入加载器**——模式依赖；`references/inputs.md`
3. **特征分析器**——音频 FFT、视频亮度或合成
4. **网格 + 渲染器**——多密度网格带位图缓存；`references/architecture.md`
5. **字符调色板**——每个项目多个；`references/architecture.md` § 调色板
6. **颜色系统**——HSV + 离散 RGB + 和谐生成；`references/architecture.md` § 颜色
7. **场景函数**——每个返回 `canvas (uint8 H,W,3)`；`references/scenes.md`
8. **色调映射**——自适应亮度归一化；`references/composition.md`
9. **着色器流水线**——`ShaderChain` + `FeedbackBuffer`；`references/shaders.md`
10. **场景表 + 分发器**——时间 → 场景函数 + 配置；`references/scenes.md`
11. **并行编码器**——N 工作线程片段渲染带 ffmpeg 管道
12. **主函数**——编排完整流水线

### 步骤 4：质量验证

- **先测试帧**：在完整渲染之前在关键时间戳渲染单帧
- **亮度检查**：`canvas.mean() > 8` 用于所有 ASCII 内容。如果暗，降低 gamma
- **视觉连贯性**：所有场景是否感觉属于同一个视频？
- **创意愿景检查**：输出是否匹配步骤 1 中的概念？如果看起来通用，返回

## 关键实现说明

### 亮度——使用 `tonemap()`，而非线性乘数

这是最常见的视觉问题。黑色背景上的 ASCII 本质上是暗的。**绝不要使用 `canvas * N` 乘数**——它们会裁剪高光。使用自适应色调映射：

```python
def tonemap(canvas, gamma=0.75):
    f = canvas.astype(np.float32)
    lo, hi = np.percentile(f[::4, ::4], [1, 99.5])
    if hi - lo < 10: hi = lo + 10
    f = np.clip((f - lo) / (hi - lo), 0, 1) ** gamma
    return (f * 255).astype(np.uint8)
```

流水线：`scene_fn() → tonemap() → FeedbackBuffer → ShaderChain → ffmpeg`

每场景 gamma：默认 0.75，solarize 0.55，posterize 0.50，亮场景 0.85。对暗层使用 `screen` 混合（而非 `overlay`）。

### 字体单元格高度

macOS Pillow：`textbbox()` 返回错误的高度。使用 `font.getmetrics()`：`cell_height = ascent + descent`。参见 `references/troubleshooting.md`。

### ffmpeg 管道死锁

绝不要对长时间运行的 ffmpeg 使用 `stderr=subprocess.PIPE`——缓冲区在 64KB 填充并死锁。重定向到文件。参见 `references/troubleshooting.md`。

### 字体兼容性

并非所有 Unicode 字符在所有字体中都能渲染。在初始化时验证调色板——渲染每个字符，检查空白输出。参见 `references/troubleshooting.md`。

### 每片段架构

对于分段视频（引用、场景、章节），将每个渲染为单独的片段文件以进行并行渲染和选择性重新渲染。参见 `references/scenes.md`。

## 创意发散（仅在用户请求实验性/创意性/独特输出时使用）

如果用户要求创意、实验性、令人惊讶或非传统的输出，选择最适合的策略并在生成代码之前推理其步骤。

- **强制连接**——当用户想要跨领域灵感时（"让它看起来有机"、"工业美学"）
- **概念混合**——当用户命名两个要组合的事物时（"海洋遇见音乐"、"空间 + 书法"）
- **间接策略**——当用户最大开放时（"给我惊喜"、"我从未见过的东西"）

### 强制连接
1. 选择一个与视觉目标无关的领域（天气系统、微生物学、建筑、流体动力学、纺织）
2. 列出其核心视觉/结构元素（侵蚀→逐渐揭示；有丝分裂→分裂复制；编织→互锁图案）
3. 将这些元素映射到 ASCII 字符和动画模式
4. 综合——"侵蚀"或"结晶"在字符网格中看起来像什么？

### 概念混合
1. 命名两个不同的视觉/概念空间（如海浪 + 乐谱）
2. 映射对应关系（波峰 = 高音，波谷 = 休止，泡沫 = 断奏）
3. 选择性地混合——保留最有趣的映射，丢弃强制的
4. 开发仅在混合中出现的涌现属性

### 间接策略
1. 抽取一个："将你的错误视为隐藏的意图" / "使用旧想法" / "你最好的朋友会做什么？" / "强调缺陷" / "把它倒过来" / "只是一部分，不是全部" / "反转"
2. 根据当前 ASCII 动画挑战解释指令
3. 在编写代码之前将横向洞察应用于视觉设计
