### 1.2 请求日志

每个请求都会被记录：

```

```

**日志包含的信息：**
- 时间戳：`2026-05-07 14:47:03,249`
- 日志级别：`[INFO]`
- 模块名：`fastapi_server.middleware.auth`
- 请求方法：`GET`
- 请求路径：`/api/v1/health`
- 请求 ID：`req_1778136423249_135285957371168`

- 请求耗时：`0.000s`
- 客户端 IP：`127.0.0.1:45564`

### 1.3 查看后台运行服务的日志

如果服务在后台运行，可以使用以下命令查看日志：

```bash
# 查看最近的日志
tail -f /var/log/hermes-agent/api.log

# 查看最近的 100 行日志
tail -n 100 /var/log/hermes-agent/api.log

# 实时查看日志（滚动）
tail -f /var/log/hermes-agent/api.log | grep "ERROR"
```

## 2. API 端点查看

### 2.1 健康检查端点

```bash
# 查看服务健康状态
curl http://localhost:8001/api/v1/health | python -m json.tool
```

**返回示例：**
```json
{
    "status": "healthy",
    "version": "1.0.0",
    "uptime": 16.32,
    "agents_active": 0,
    "sessions_count": 0,
    "timestamp": "2026-05-07T14:47:03.249582"
}
```

**字段说明：**
- `status`: 服务状态（healthy/unhealthy）
- `version`: 服务版本
- `uptime`: 运行时间（秒）
- `agents_active`: 活跃的 Agent 数量
- `sessions_count`: 会话数量
- `timestamp`: 检查时间戳

### 2.2 就绪检查端点

```bash
# 查看服务是否准备好接收请求
curl http://localhost:8001/api/v1/ready | python -m json.tool
```

**返回示例：**
```json
{
    "status": "ready",
    "version": "1.0.0",
    "uptime": 120.5,
    "agents_active": 0,
    "database": "connected",
    "redis": "unknown",
    "llm_provider": "available",
    "timestamp": "2026-05-07T14:49:03.249582"
}
```

**字段说明：**
- `status`: 就绪状态（ready/not_ready）
- `database`: 数据库连接状态
- `redis`: Redis 连接状态（如果配置）
- `llm_provider`: LLM 提供商 API 状态

### 2.3 根路径端点

```bash
# 查看服务基本信息
curl http://localhost:8001/ | python -m json.tool
```

**返回示例：**
```json
{
    "name": "Hermes-Agent API",
    "version": "1.0.0",
    "docs": "/docs",
    "health": "/api/v1/health"
}
```

### 2.4 配置信息端点

```bash
# 查看脱敏后的服务配置
curl http://localhost:8001/api/v1/config | python -m json.tool
```

**返回示例：**
```json
{
    "model": "dashscope/deepseek-v4-flash",
    "max_iterations": 90,
    "provider": "dashscope",
    "version": "1.0.0",
```

### 2.5 交互式 API 文档

在浏览器中打开以下地址查看完整的 API 文档：

- **Swagger UI**: http://localhost:8001/docs
- **ReDoc**: http://localhost:8001/redoc
- **OpenAPI JSON**: http://localhost:8001/openapi.json

## 3. 进程状态查看

### 3.1 查看服务进程

```bash
# 查看运行的 Python 进程
ps aux | grep "uvicorn fastapi_server.main"

# 查看占用 8001 端口的进程
lsof -i :8001

# 查看进程的详细信息
ps -p <PID> -o pid,ppid,user,%cpu,%mem,etime,cmd
```

### 3.2 查看端口占用

```bash
# 查看 8001 端口的占用情况
netstat -tlnp | grep 8001

# 或使用 ss 命令
ss -tlnp | grep 8001
```

**输出示例：**
```
tcp        0      0 0.0.0.0:8001    0.0.0.0:*    LISTEN   2783615/python3
```

### 3.3 查看系统资源使用

```bash
# 查看进程的 CPU 和内存使用
top -p <PID>

# 查看进程的内存使用详情
cat /proc/<PID>/status | grep -E "VmRSS|VmSize"

# 查看进程打开的文件
lsof -p <PID>
```

## 4. 日志文件查看

### 4.1 配置日志文件

在 `.env` 文件中配置日志文件路径：

```ini
# .env
LOG_LEVEL=INFO
LOG_FILE=/var/log/hermes-agent/api.log
LOG_FORMAT=json
```

### 4.2 查看日志文件

```bash
# 创建日志目录
sudo mkdir -p /var/log/hermes-agent
sudo chown $USER:$USER /var/log/hermes-agent

# 查看日志文件
tail -f /var/log/hermes-agent/api.log

# 搜索错误日志
grep "ERROR" /var/log/hermes-agent/api.log

# 查看今天的日志
grep "$(date +%Y-%m-%d)" /var/log/hermes-agent/api.log
```

### 4.3 日志轮转

配置日志轮转防止日志文件过大：

```bash
# 创建日志轮转配置
sudo vim /etc/logrotate.d/hermes-agent
```

**配置内容：**
```
/var/log/hermes-agent/*.log {
    daily
    rotate 7
    compress
    delaycompress
    missingok
    notifempty
    create 0640 $USER $USER
    postrotate
        kill -HUP `cat /var/run/hermes-agent.pid 2>/dev/null` 2>/dev/null || true
    endscript
}
```

## 5. Prometheus 监控指标

### 5.1 查看 Prometheus 指标

```bash
# 获取 Prometheus 格式的指标
curl http://localhost:8001/api/v1/metrics
```

**输出示例：**
```
# HELP hermes_requests_total Total number of requests
# TYPE hermes_requests_total counter
hermes_requests_total{endpoint="/api/v1/health",method="GET",status_code="200"} 1.0

# HELP hermes_request_duration_seconds Request duration in seconds
# TYPE hermes_request_duration_seconds histogram
hermes_request_duration_seconds_bucket{endpoint="/api/v1/health",le="0.1"} 1.0
hermes_request_duration_seconds_bucket{endpoint="/api/v1/health",le="0.5"} 1.0
hermes_request_duration_seconds_sum{endpoint="/api/v1/health"} 0.001

# HELP hermes_agents_active Number of active agents
# TYPE hermes_agents_active gauge
hermes_agents_active 0.0
```

### 5.2 指标类型说明

**请求计数器（Counter）：**
- `hermes_requests_total`: 总请求数
  - 标签：`endpoint`（端点）、`method`（方法）、`status_code`（状态码）

**请求延迟直方图（Histogram）：**
- `hermes_request_duration_seconds`: 请求延迟（秒）
  - 标签：`endpoint`（端点）
  - 桶：0.1, 0.5, 1.0, 2.0, 5.0, 10.0, 30.0, 60.0, 120.0 秒

**活跃 Agent 仪表（Gauge）：**
- `hermes_agents_active`: 当前活跃的 Agent 数量

### 5.3 配置 Prometheus 监控

在 Prometheus 配置中添加：

```yaml
# prometheus.yml
scrape_configs:
  - job_name: 'hermes-agent'
    static_configs:
      - targets: ['localhost:8001']
    metrics_path: '/api/v1/metrics'
    scrape_interval: 15s
```

### 5.4 Grafana 仪表盘

可以使用以下指标创建 Grafana 仪表盘：

**请求速率：**
```promql
rate(hermes_requests_total[5m])
```

**请求延迟：**
```promql
histogram_quantile(0.95, rate(hermes_request_duration_seconds_bucket[5m]))
```

**错误率：**
```promql
sum(rate(hermes_requests_total{status_code=~"5.."}[5m])) / sum(rate(hermes_requests_total[5m]))
```

**活跃 Agent 数：**
```promql
hermes_agents_active
```

## 6. 常见问题排查

### 6.1 服务无法启动

**问题：** 端口已被占用
```
ERROR: [Errno 98] error while attempting to bind on address ('0.0.0.0', 8001): address already in use
```

**解决方法：**
```bash
# 查找占用端口的进程
lsof -i :8001

# 杀死占用的进程
lsof -ti:8001 | xargs kill -9

# 或者更改端口
uvicorn fastapi_server.main:app --host 0.0.0.0 --port 8002
```

### 6.2 服务响应慢

**查看请求延迟：**
```bash
# 查看日志中的请求耗时
grep "Duration" /var/log/hermes-agent/api.log | tail -20
```

**查看系统负载：**
```bash
# 查看系统负载
uptime

# 查看内存使用
free -h

# 查看磁盘 IO
iostat -x 1 5
```

### 6.3 服务崩溃

**查看崩溃前的日志：**
```bash
# 查看最近的错误日志
tail -n 100 /var/log/hermes-agent/api.log | grep ERROR

# 查看系统日志
journalctl -u hermes-agent -n 50
```

**启用调试模式：**
```ini
# .env
DEBUG=true
LOG_LEVEL=DEBUG
```

## 7. 监控最佳实践

### 7.1 设置告警

在 Prometheus AlertManager 中配置告警：

```yaml
# alertmanager.yml
groups:
  - name: hermes-agent
    rules:
      - alert: HighErrorRate
        expr: sum(rate(hermes_requests_total{status_code=~"5.."}[5m])) / sum(rate(hermes_requests_total[5m])) > 0.05
        for: 5m
        labels:
          severity: critical
        annotations:
          summary: "Hermes-Agent 错误率过高"
          description: "错误率超过 5%"

      - alert: HighLatency
        expr: histogram_quantile(0.95, rate(hermes_request_duration_seconds_bucket[5m])) > 5
        for: 5m
        labels:
          severity: warning
        annotations:
          summary: "Hermes-Agent 响应延迟过高"
          description: "P95 延迟超过 5 秒"
```

### 7.2 日志聚合

使用 ELK Stack（Elasticsearch, Logstash, Kibana）或 Loki 进行日志聚合：

```yaml
# filebeat.yml
filebeat.inputs:
  - type: log
    enabled: true
    paths:
      - /var/log/hermes-agent/*.log
    json.keys_under_root: true
    json.add_error_key: true

output.logstash:
  hosts: ["localhost:5044"]
```

### 7.3 分布式追踪

集成 Jaeger 或 Zipkin 进行分布式追踪：

```python
# 在 main.py 中添加
from opentelemetry import trace
from opentelemetry.exporter.jaeger.thrift import JaegerExporter
from opentelemetry.sdk.trace import TracerProvider
from opentelemetry.sdk.trace.export import BatchSpanProcessor

# 配置 Jaeger exporter
jaeger_exporter = JaegerExporter(
    agent_host_name="localhost",
    agent_port=6831,
)

trace.get_tracer_provider().add_span_processor(
    BatchSpanProcessor(jaeger_exporter)
)
```

## 总结

查看 FastAPI 服务运行信息的主要方式：

1. **实时日志**：直接查看终端输出或使用 `tail -f` 查看日志文件
2. **API 端点**：通过 `/api/v1/health`、`/api/v1/ready`、`/api/v1/config` 等端点查看状态
3. **进程监控**：使用 `ps`、`lsof`、`top` 等命令查看进程状态
4. **Prometheus 指标**：通过 `/api/v1/metrics` 获取监控指标
5. **API 文档**：访问 `/docs` 或 `/redoc` 查看完整的 API 文档

**推荐监控组合：**
- 实时日志 + Prometheus + Grafana + 告警
- 日志聚合（ELK/Loki）+ 分布式追踪（Jaeger）
- 健康检查 + 就绪检查（Kubernetes）

这样可以全面了解服务的运行状态，及时发现和解决问题。
