## 配置项详解

### 1. `enable_metrics: bool = True`

**作用：** 启用/禁用 Prometheus 指标收集

```python
# 启用指标收集（生产环境推荐）
enable_metrics = True

# 禁用指标收集（开发环境可关闭）
enable_metrics = False
```

**配置说明：**
- `True`：启用指标收集，暴露 `/metrics` 端点
- `False`：禁用指标收集，减少性能开销
- 生产环境：**必须启用**
- 开发环境：可选禁用

---

### 2. `prometheus_port: int = 9090`

**作用：** Prometheus 指标服务器端口

```python
# 默认端口
prometheus_port = 9090

# 自定义端口
prometheus_port = 8001

# 与主服务同端口（通过路径区分）
prometheus_port = 8000  # /metrics 路径
```

**配置说明：**
- 默认值：9090（Prometheus 标准端口）
- 监听地址：`0.0.0.0:{prometheus_port}`
- 指标端点：`http://localhost:9090/metrics`
- 安全建议：生产环境应限制访问 IP

---

### 3. `sentry_dsn: Optional[str] = None`

**作用：** Sentry 错误追踪服务 DSN（Data Source Name）

```python
# 不启用 Sentry
sentry_dsn = None

# 启用 Sentry（替换为你的 DSN）
sentry_dsn = "https://abc123@o123456.ingest.sentry.io/1234567"

# 从环境变量读取
sentry_dsn = os.getenv("SENTRY_DSN")
```

**配置说明：**
- `None`：不启用 Sentry 错误追踪
- DSN 格式：`https://<public_key>@<host>/<project_id>`
- 获取方式：在 Sentry .io 创建项目后获取
- 生产环境：**强烈建议启用**

### 指标类型

Prometheus 支持 4 种核心指标类型：

#### 1. Counter（计数器）

**特点：** 只增不减，用于累计值

```python
from prometheus_client import Counter

# 定义指标
REQUEST_COUNT = Counter(
    "http_requests_total",      # 指标名称
    "Total HTTP requests",      # 描述
    ["method", "endpoint", "status"]  # 标签
)

# 使用
REQUEST_COUNT.labels(
    method="POST",
    endpoint="/api/v1/chat/completions",
    status="200"
).inc()

# 查询
# 总请求数
sum(http_requests_total)

# 按状态码分组
sum by (status) (http_requests_total)

# 请求速率
rate(http_requests_total[5m])
```

**适用场景：**
- 请求总数
- 错误总数
- 任务完成数

---

#### 2. Gauge（仪表盘）

**特点：** 可增可减，用于瞬时值

```python
from prometheus_client import Gauge

# 定义指标
ACTIVE_CONNECTIONS = Gauge(
    "active_connections",  # 指标名称
    "Number of active connections"  # 描述
)

# 使用
ACTIVE_CONNECTIONS.set(42)  # 设置值
ACTIVE_CONNECTIONS.inc()    # +1
ACTIVE_CONNECTIONS.dec()    # -1

# 查询
# 当前连接数
active_connections

# 最大值
max_over_time(active_connections[1h])

# 平均值
avg_over_time(active_connections[1h])
```

**适用场景：**
- 活跃连接数
- CPU 使用率
- 内存使用率
- 队列长度

---

#### 3. Histogram（直方图）

**特点：** 统计分布，用于延迟、大小

```python
from prometheus_client import Histogram

# 定义指标
REQUEST_DURATION = Histogram(
    "http_request_duration_seconds",  # 指标名称
    "HTTP request duration in seconds",  # 描述
    ["method", "endpoint"],  # 标签
    buckets=[0.005, 0.01, 0.025, 0.05, 0.1, 0.25, 0.5, 1.0, 2.5, 5.0, 10.0]  # 桶
)

# 使用
@REQUEST_DURATION.time()  # 装饰器自动计时
async def handle_request():
    ...

# 查询
# P50 延迟（中位数）
histogram_quantile(0.50, rate(http_request_duration_seconds_bucket[5m]))

# P95 延迟
histogram_quantile(0.95, rate(http_request_duration_seconds_bucket[5m]))

# P99 延迟
histogram_quantile(0.99, rate(http_request_duration_seconds_bucket[5m]))

# 平均延迟
rate(http_request_duration_seconds_sum[5m]) / rate(http_request_duration_seconds_count[5m])
```

**适用场景：**
- 请求延迟
- 响应大小
- 处理时间

---

#### 4. Summary（摘要）

**特点：** 客户端计算分位数，精度高但无法聚合

```python
from prometheus_client import Summary

# 定义指标
REQUEST_LATENCY = Summary(
    "request_latency_seconds",
    "Request latency in seconds",
    ["endpoint"]
)

# 使用
@REQUEST_LATENCY.time()
async def handle_request():
    ...

# 查询
# P95 延迟（在客户端计算）
request_latency_seconds{quantile="0.95"}
```

**与 Histogram 的区别：**
- Histogram：服务端计算分位数，可聚合，推荐
- Summary：客户端计算分位数，精度高，不可聚合

### 指标收集实现

#### 中间件方式

```python
from prometheus_client import Counter, Histogram
import time
from fastapi import Request

# 定义指标
REQUEST_COUNT = Counter(
    "http_requests_total",
    "Total HTTP requests",
    ["method", "endpoint", "status"],
)

REQUEST_DURATION = Histogram(
    "http_request_duration_seconds",
    "HTTP request duration",
    ["method", "endpoint"],
    buckets=[0.01, 0.05, 0.1, 0.5, 1.0, 2.0, 5.0],
)

class MetricsMiddleware:
    def __init__(self, app):
        self.app = app
    
    async def __call__(self, scope, receive, send):
        if scope["type"] != "http":
            return await self.app(scope, receive, send)
        
        request = Request(scope, receive)
        start_time = time.time()
        
        # 捕获响应
        async def send_wrapper(message):
            if message["type"] == "http.response.start":
                status_code = message["status"]
                
                # 记录指标
                elapsed = time.time() - start_time
                REQUEST_COUNT.labels(
                    method=request.method,
                    endpoint=request.url.path,
                    status=status_code
                ).inc()
                
                REQUEST_DURATION.labels(
                    method=request.method,
                    endpoint=request.url.path
                ).observe(elapsed)
            
            await send(message)
        
        await self.app(scope, receive, send_wrapper)

# 使用
app.add_middleware(MetricsMiddleware)
```

#### 装饰器方式

```python
from functools import wraps
import time

def track_metrics(metric_name: str, labels: dict = None):
    """指标追踪装饰器"""
    def decorator(func):
        @wraps(func)
        async def wrapper(*args, **kwargs):
            start_time = time.time()
            
            try:
                result = await func(*args, **kwargs)
                
                # 记录成功
                AGENT_RUNS.labels(
                    model=kwargs.get("model", "unknown"),
                    status="success"
                ).inc()
                
                return result
                
            except Exception as e:
                # 记录失败
                AGENT_RUNS.labels(
                    model=kwargs.get("model", "unknown"),
                    status="error"
                ).inc()
                raise
            
            finally:
                # 记录耗时
                elapsed = time.time() - start_time
                AGENT_DURATION.observe(elapsed)
        
        return wrapper
    return decorator

# 使用
@track_metrics("agent_run")
async def run_agent(message: str, model: str):
    ...
```

#### 后台任务方式

```python
import asyncio
from prometheus_client import Gauge

# 定义指标
CPU_USAGE = Gauge("hermes_cpu_usage_percent", "CPU usage")
MEMORY_USAGE = Gauge("hermes_memory_usage_bytes", "Memory usage")

async def collect_system_metrics():
    """后台收集系统指标"""
    import psutil
    
    while True:
        # CPU 使用率
        cpu_percent = psutil.cpu_percent(interval=1)
        CPU_USAGE.set(cpu_percent)
        
        # 内存使用
        memory = psutil.virtual_memory()
        MEMORY_USAGE.set(memory.used)
        
        # 等待 15 秒
        await asyncio.sleep(15)

# 启动后台任务
@app.on_event("startup")
async def startup():
    asyncio.create_task(collect_system_metrics())
```

```python
from starlette.middleware.base import BaseHTTPMiddleware
from starlette.responses import PlainTextResponse

class MetricsAuthMiddleware(BaseHTTPMiddleware):
    async def dispatch(self, request, call_next):
```

### 集成配置

```python
import sentry_sdk
from sentry_sdk.integrations.fastapi import FastApiIntegration
from sentry_sdk.integrations.logging import LoggingIntegration

def init_sentry(dsn: str, environment: str = "production"):
    """初始化 Sentry"""
    
    sentry_sdk.init(
        dsn=dsn,
        environment=environment,
        
        # 集成
        integrations=[
            FastApiIntegration(),
            LoggingIntegration(),
        ],
        
        # 性能监控（可选）
        traces_sample_rate=0.1,  # 10% 采样率
        
        # 错误采样
        sample_rate=1.0,  # 100% 错误上报
        
        # 发送上下文
        send_default_pii=True,  # 包含用户信息
        
        # 过滤敏感信息
        before_send=filter_sensitive_data,
    )

def filter_sensitive_data(event, hint):
    """过滤敏感数据"""
    # 移除密码字段
    if "request" in event:
        if "data" in event["request"]:
            data = event["request"]["data"]
            if "password" in data:
                data["password"] = "[REDACTED]"
    
    return event

# 使用
from fastapi_server.config import settings

if settings.sentry_dsn:
    init_sentry(
        dsn=settings.sentry_dsn,
        environment="production" if not settings.debug else "development"
    )
```

### 错误捕获

```python
import sentry_sdk
from fastapi import HTTPException

# 自动捕获异常
@app.exception_handler(Exception)
async def global_exception_handler(request, exc):
    """全局异常处理器"""
    
    # 上报 Sentry
    sentry_sdk.capture_exception(exc)
    
    # 返回友好错误
    if isinstance(exc, HTTPException):
        return {
            "error": "Request failed",
            "detail": exc.detail,
            "status_code": exc.status_code,
        }
    else:
        return {
            "error": "Internal server error",
            "status_code": 500,
        }

# 手动捕获
async def process_user_data(data: dict):
    try:
        # 业务逻辑
        ...
    except ValidationError as e:
        # 记录错误上下文
        sentry_sdk.set_context("user_data", {
            "user_id": data.get("user_id"),
            "operation": "validate",
        })
        sentry_sdk.set_tag("error_type", "validation")
        sentry_sdk.capture_exception(e)
        raise
```

### 上下文增强

```python
import sentry_sdk

class SentryContextMiddleware:
    """Sentry 上下文中间件"""
    
    async def __call__(self, request, call_next):
        # 设置用户上下文
        user_id = request.headers.get("X-User-ID")
        if user_id:
            sentry_sdk.set_user({"id": user_id})
        
        # 设置请求上下文
        sentry_sdk.set_context("request", {
            "method": request.method,
            "url": str(request.url),
            "path": request.url.path,
        })
        
        # 设置标签
        sentry_sdk.set_tag("endpoint", request.url.path)
        sentry_sdk.set_tag("method", request.method)
        
        # 添加 Breadcrumb（面包屑）
        sentry_sdk.add_breadcrumb(
            category="http",
            message=f"{request.method} {request.url.path}",
            level="info",
        )
        
        try:
            response = await call_next(request)
            
            # 记录响应状态
            sentry_sdk.add_breadcrumb(
                category="http",
                message=f"Response: {response.status_code}",
                level="info",
            )
            
            return response
            
        except Exception as e:
            # 错误已自动捕获
            raise

# 使用
app.add_middleware(SentryContextMiddleware)
```

```python
        return response

# 数据库查询追踪
async def query_database(sql: str):
    with start_transaction(
        op="db.query",
        name="Database Query"
    ) as span:
        span.set_data("db.system", "sqlite")
        span.set_data("db.statement", sql)
        
        result = await db.execute(sql)
        
        span.set_data("db.rows_affected", result.rowcount)
        
        return result
```

## 总结

### 核心要点

1. **Prometheus 指标**：
   - Counter（计数器）：请求总数、错误总数
   - Gauge（仪表盘）：CPU 使用率、内存使用率
   - Histogram（直方图）：请求延迟、响应大小

2. **Sentry 错误追踪**：
   - 自动异常捕获
   - 完整堆栈信息
   - 用户上下文追踪

3. **核心监控指标**：
   - RED 方法：Rate、Errors、Duration
   - USE 方法：Utilization、Saturation、Errors
   - 业务指标：Agent 效率、Token 使用

4. **告警设计**：
   - P0 紧急：服务宕机、高错误率
   - P1 严重：高 CPU、高内存
   - P2 警告：队列过长、工具失败
   - P3 提示：高成本、低磁盘
