# Hermes-Agent 模型工具模块架构分析

## 1. 系统概述

`model_tools.py` 是 Hermes-Agent 的**工具编排层（Tool Orchestration Layer）**，位于工具注册表（`tools.registry`）与上层调用者（`run_agent.py`、`cli.py`、`batch_runner.py`、RL 环境）之间。该模块负责：

- **工具发现**：自动导入所有工具模块，触发注册
- **Schema 过滤**：基于工具集（toolset）启用/禁用工具
- **参数类型转换**：将 LLM 返回的字符串参数转换为正确的 JSON Schema 类型
- **异步桥接**：为同步调用者提供异步工具执行能力
- **工具调度**：将函数调用分派到正确的工具处理器

### 1.1 核心功能特性

| 功能模块 | 描述 |
|---------|------|
| **工具发现** | 自动导入 20+ 个工具模块，触发 `registry.register()` |
| **MCP 发现** | 从配置中加载外部 MCP 服务器工具 |
| **插件发现** | 加载用户/项目/pip 插件工具 |
| **工具集过滤** | 基于 `enabled_toolsets` / `disabled_toolsets` 过滤工具 |
| **Schema 动态生成** | 为 `execute_code` 动态生成可用工具列表 |
| **参数类型强制转换** | 将字符串 `"42"` → `42`，`"true"` → `true` |
| **异步桥接** | 为同步上下文提供持久事件循环 |
| **并发安全** | 主线程与工作线程各自拥有独立的事件循环 |
