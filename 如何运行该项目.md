## 🔑 配置 API 密钥

Hermes 支持多种 LLM 提供商，选择其中一个配置即可。

### 方式 1: 使用 `hermes login` 命令（推荐）

```bash
# 登录 OpenRouter（推荐，支持多种模型）
hermes login openrouter

# 登录 Anthropic
hermes login anthropic

# 登录 OpenAI
hermes login openai

# 登录阿里云（通义千问）
hermes login alibaba
```

### 方式 2: 编辑 `.env` 文件

```bash
# 创建/编辑配置文件
nano ~/.hermes/.env
```

添加以下内容（选择你使用的提供商）：

```bash
# OpenRouter (推荐 - 支持 Claude/GPT/多种模型)
OPENROUTER_API_KEY=your_openrouter_key_here

# Anthropic (Claude)
ANTHROPIC_API_KEY=sk-ant-...

# OpenAI (GPT)
OPENAI_API_KEY=sk-...

# Google (Gemini)
GEMINI_API_KEY=...

# 阿里云 DashScope (通义千问)
DASHSCOPE_API_KEY=sk-...
DASHSCOPE_BASE_URL=https://dashscope.aliyuncs.com/compatible-mode/v1
```

### 方式 3: 使用环境变量

```bash
export OPENROUTER_API_KEY=your_key_here
export ANTHROPIC_API_KEY=sk-ant-...
```

### 获取 API 密钥

| 提供商 | 获取地址 | 免费额度 |
|--------|---------|---------|
| **OpenRouter** | https://openrouter.ai/keys | ✅ 有免费模型 |
| **Anthropic** | https://console.anthropic.com/ | ❌ 需要付费 |
| **OpenAI** | https://platform.openai.com/api-keys | ❌ 需要付费 |
| **Google Gemini** | https://aistudio.google.com/app/apikey | ✅ 有免费额度 |
| **阿里云** | https://dashscope.console.aliyun.com/ | ✅ 有免费额度 |

## 💻 运行方式

### 方式 1: 交互式 CLI（推荐）

```bash
hermes
```

**功能**:
- ✅ 美丽的终端界面
- ✅ 多行编辑支持
- ✅ 斜杠命令自动补全
- ✅ 实时工具输出
- ✅ 会话历史管理

**示例会话**:
```
> Hello! Can you help me write a Python script?

Hello! I'd be happy to help you write a Python script. What would you like the script to do?
```

### 方式 2: 单次查询模式

```bash
# 简单查询
hermes chat -q "What is the capital of France?"

# 复杂任务
hermes chat -q "帮我创建一个 Python 文件，实现批量重命名功能"
```

### 方式 3: 恢复会话

```bash
# 恢复最近的会话
hermes --continue

# 恢复特定会话
hermes --continue "my-project"

# 通过会话 ID 恢复
hermes --resume 20260421_171748_d7bd14
```

### 方式 4: 消息网关（多平台）

```bash
# 启动 Telegram 机器人
hermes gateway telegram

# 启动 Discord 机器人
hermes gateway discord

# 启动所有配置的平台
hermes gateway start
```

### 方式 5: 编辑器集成

```bash
# VS Code / Zed / JetBrains 集成
hermes acp
```

## ⚙️ 配置模型

### 当前配置（阿里云 GLM-5）

```yaml
Provider: alibaba
Model: qwen-plus (通义千问 Plus)
Base URL: https://dashscope.aliyuncs.com/compatible-mode/v1
```

### 切换模型

```bash
# 交互式选择
hermes model

# 或直接指定
/model anthropic/claude-3.5-sonnet
/model openai/gpt-4o
/model openrouter/anthropic/claude-3.5-sonnet
/model gemini/gemini-1.5-pro
/model qwen-plus
```

### 推荐模型

| 场景 | 推荐模型 | 说明 |
|------|---------|------|
| **日常使用** | `qwen-plus` | 阿里云通义千问，性价比高 |
| **最强性能** | `anthropic/claude-3.5-sonnet` | Claude 3.5 Sonnet |
| **代码能力** | `openai/gpt-4o` | GPT-4o |
| **免费额度** | `gemini/gemini-1.5-pro` | Google Gemini |
| **本地运行** | `ollama/llama3.2:latest` | Ollama 本地模型 |

## 🎯 使用技能

技能是预定义的程序化指令，让 Agent 执行特定任务。

### 浏览技能

```bash
/skills
```

**输出示例**:
```
Available Skills:
  📦 autonomous-ai-agents: claude-code, codex, hermes-agent, opencode
  🎨 creative: architecture-diagram, ascii-art, ascii-video, ...
  💻 devops: commit-worknotes, git-commit-push, ...
  🐙 github: codebase-inspection, github-auth, ...
  ...
```

### 安装技能

```bash
# 安装 GitHub 认证技能
/skills install github-auth

# 安装代码审查技能
/skills install requesting-code-review
```

### 使用技能

安装后，技能会自动集成到对话中：

```
> 帮我检查这个仓库的代码质量

[自动使用 github-auth 和 codebase-inspection 技能]
```

### 创建自定义技能

在 `~/.hermes/skills/` 目录下创建技能：

```bash
mkdir -p ~/.hermes/skills/my-skill
cat > ~/.hermes/skills/my-skill/SKILL.md << 'EOF'
## Name
my-custom-skill

## Description
What your skill does

## Triggers
- trigger pattern 1
- trigger pattern 2

## Instructions
Detailed instructions for the AI
EOF
```

## 📱 消息平台

### Telegram

**设置步骤**:

1. **创建机器人**:
   - 在 Telegram 中联系 [@BotFather](https://t.me/BotFather)
   - 发送 `/newbot`
   - 按照提示设置名称和用户名
   - 获取 bot token

2. **配置 Hermes**:
```bash
hermes gateway setup telegram
# 输入 bot token
```

3. **启动网关**:
```bash
hermes gateway telegram
```

4. **开始聊天**:
   - 在 Telegram 中搜索你的机器人
   - 发送 `/start`
   - 开始对话！

### Discord

**设置步骤**:

1. **创建 Discord 应用**:
   - 访问 [Discord Developer Portal](https://discord.com/developers/applications)
   - 创建新应用
   - 获取 bot token

2. **邀请机器人**:
```bash
hermes gateway discord invite
# 跟随指引邀请到服务器
```

3. **配置并启动**:
```bash
hermes gateway setup discord
hermes gateway discord
```

### 其他平台

```bash

# WhatsApp
hermes gateway whatsapp

# Signal
hermes gateway signal

# Home Assistant
```

## 🔧 故障排除

### 常见问题

#### 1. "No API key found"

**解决方案**:
```bash
# 检查 .env 文件
cat ~/.hermes/.env

# 重新登录
hermes login openrouter

# 或设置环境变量
export OPENROUTER_API_KEY=your_key_here
```

#### 2. "Module not found"

**解决方案**:
```bash
# 确认虚拟环境已激活
source venv/bin/activate

# 重新安装依赖
pip install -e ".[all]"
```

#### 3. "Gateway failed to start"

**解决方案**:
```bash
# 检查平台配置
hermes gateway status

# 查看详细日志
hermes logs

# 重新配置网关
hermes gateway setup telegram
```

#### 4. 命令执行失败

**解决方案**:
```bash
# 检查工具是否启用
hermes tools list

# 检查工具依赖
hermes doctor

# 查看工具日志
hermes logs --tool terminal
```

### 诊断命令

```bash
# 全面检查
hermes doctor

# 查看日志
hermes logs

# 查看配置
hermes config

# 查看会话
hermes sessions list

# 查看使用统计
hermes insights --days 7
```
